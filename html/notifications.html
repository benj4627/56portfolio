NOTIFIKATIONSSIDE

KOMPONENT

  • HTML
  • CSS
  • JavaScript

Jeg udvikler en notifikationsside baseret på en eksisterende styleguide og statisk mockup, hvilket sikrer, at den fungerer som en integreret del af et større website. Gennem målrettet brug af vanilla JavaScript, semantisk HTML og genanvendelig CSS opnår jeg en funktionel, tilgængelig og intuitiv løsning, der er optimeret til en ensartet oplevelse på desktop og mobil.

    OVERSIGT

  • HTML
  • CSS
  • JavaScript
  • Begrænsninger

HTML

I processen opsætter jeg et kodemiljø, hvor jeg klargør de korrekte filer til typografi, HTML, CSS og JavaScript samt grundlæggende styling.

Jeg opretter overblik over HTML-strukturen ved at opdele dokumentet i relevante tags som body, main og sections. Hvis projektet var fuldt realiseret, ville det også inkludere header og footer. Jeg færdiggør en section ad gangen med HTML og CSS. Billederne viser korrekt semantisk HTML, der fremmer webtilgængelighed, herunder alt-tekster samt title og meta description i head-sektionen.

CSS

Ved opsætning af kodemiljøet klargør jeg CSS-dokumentet med et reset af base styling for at undgå konflikter. Jeg opsætter CSS-variabler til farver, spacing og border-radius, importerer fonts og anvender basal styling på typografi, billeder og knapper.

CSS-variablerne sikrer konsistens i projektet og gør det nemt at ændre farver, spacing eller border-radius i hele dokumentet, hvis der skal produceres en større grænseflade. Dette bidrager til en effektiv arbejdsproces gennem hele projektet.

Efter opsætning af CSS programmerer jeg en media query for korrekt visning på mobil, der komprimerer containeren til en passende bredde ved hjælp af flex-direction og grid. Jeg tilføjer også på interaktive elementer for at forbedre brugervenligheden med tydelig visuel feedback.

JavaScript

For at tilføje funktionalitet bruger jeg vanilla JavaScript. Først fanger jeg de relevante HTML-elementer fra DOM’en med document.querySelector og querySelectorAll, da jeg nogle gange har brug for at ramme flere elementer med samme class og loope igennem dem med forEach. I tilfælde, hvor kun ét element skal bruges, er querySelector tilstrækkelig.

Den første funktions formål er at loope igennem ulæste notifikationer, når buttonEl trykkes, opdatere stylingen, fjerne de røde prikker ved at tilføje klassen hidden og opdatere tekstindholdet i numberEl for korrekt at afspejle antallet af ulæste notifikationer.

Den sidste del af JavaScriptet tilføjer en eventListener til hver notifikation, så man kan markere dem som "læst" ved at klikke på dem. Når en notifikation klikkes, ændres stylingen til "læst" ved at fjerne den blå baggrund og den røde prik. Herefter tæller koden de ulæste notifikationer igen og opdaterer antallet i "notifications" i toppen af siden.

Padding-klassen adskiller de læste og ulæste notifikationer og gør det muligt at opdatere tælleren korrekt. Notifikationer med klassen padding tælles ikke med som ulæste. document.querySelectorAll('.notification:not(.padding)') vælger derfor kun de notifikationer uden padding-klassen, hvilket holder styr på antallet af ulæste notifikationer.

BEGRÆNSINGER

Siden fungerer som en del af en større grænseflade, men opgaven er begrænset til notifikationssiden. CSS-variabler er opsat for at sikre nem konsistens ved en fremtidig udvidelse af projektet. Da siden eksisterer alene, fører links ikke til specifikke indlæg eller billeder. Opgaven følger også den udleverede styleguide for at nå så tæt som muligt på den givne mockup.

Funktionaliteten kunne muligvis have været mere effektiv med Vue.js, hvor data binding med { ref } kunne udnyttes. Desuden ville koden være nemmere at vedligeholde, hvis projektet voksede, da den så ville være mere overskuelig.
